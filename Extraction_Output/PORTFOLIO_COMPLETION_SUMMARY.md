### Challenges Encountered

‚ö†Ô∏è **Data Availability:**
- Unit counts frequently missing from contracts
- Service specifications often incomplete in invoice data
- Tonnage data rarely available in invoices
- Pickup frequency not documented in billing records

‚ö†Ô∏è **Data Consistency:**
- Different invoice formats by vendor and region
- Varying levels of detail in contracts
- Some properties have extensive history, others minimal
- Arizona Excel format vs. other formats

‚ö†Ô∏è **Technical:**
- Unicode encoding issues in Windows console (resolved with UTF-8)
- Large dataset processing for some properties
- Chart.js integration in self-contained HTML
- Excel formula complexity in multi-sheet workbooks

### Best Practices Established

‚úÖ **For Future Analyses:**
1. Always validate unit counts from official property records FIRST
2. Request service specifications from vendors upfront (don't rely on invoices)
3. Establish direct vendor API connections for ongoing data collection
4. Implement automated monthly data ingestion
5. Maintain calculation reference documentation as single source of truth
6. Flag limitations prominently rather than making assumptions
7. Provide data collection plans for missing information
8. Use consistent deliverable templates across portfolio
